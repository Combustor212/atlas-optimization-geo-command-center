# Explicit MEO Transparency Implementation

## Overview
Implemented three explicit backend objects that the frontend displays **verbatim** to guarantee single source of truth for MEO scoring explanations.

## Backend Changes

### 1. New Schema Types (`mgo-scanner-backend/src/meo/meoSchema.ts`)

#### `MEOInputsUsed` - Raw GBP fields used for scoring
```typescript
export interface MEOInputsUsed {
  rating: number;
  totalReviews: number;
  photoCount: number;
  hasWebsite: boolean;
  hasPhone: boolean;
  hasHours: boolean;
  hasDescription: boolean;
  hasOwnerResponses: boolean;
  reviewResponseRate: number;
  completenessScore: number;
  categoryCanonical: string;
  isFranchise: boolean;
  isMajorNationalFranchise: boolean;
  isRegionalFranchise: boolean;
}
```

#### `MEOBreakdown` - Points per component + cap info
```typescript
export interface MEOBreakdown {
  scoringVersion: string;
  baseScore: number;
  ratingPoints: number;
  reviewPoints: number;
  photoPoints: number;
  infoPoints: number;
  engagementPoints: number;
  competitivePoints: number;
  franchiseBoostPoints: number;
  rawScoreBeforeCap: number;
  reviewReliabilityCap: number | null;
  wasCapped: boolean;
  capReason: string | null;
  finalScore: number;
}
```

#### `meoWhy` - Human-readable bullet list (string[])
Array of explanations generated by backend, including:
- Cap notification (first if applicable)
- Rating with points gained/penalty
- Reviews with points and cap mention
- Photos with points
- Profile completeness with missing fields
- Competitive position
- Franchise boost (if applicable)

### 2. Engine Implementation (`mgo-scanner-backend/src/meo/meoEngine.ts`)

Added logic to construct all three objects before returning:

```typescript
// 1) meoInputsUsed - All raw GBP fields
const meoInputsUsed = {
  rating,
  totalReviews,
  photoCount,
  hasWebsite,
  hasPhone,
  hasHours,
  hasDescription,
  hasOwnerResponses,
  reviewResponseRate,
  completenessScore,
  categoryCanonical: categoryResult.categoryCanonical,
  isFranchise: franchiseResult.isFranchise,
  isMajorNationalFranchise: franchiseResult.isMajorNationalFranchise,
  isRegionalFranchise: franchiseResult.franchiseType === 'regional'
};

// 2) meoBreakdown - Points per component
const meoBreakdown = {
  scoringVersion: SCORING_VERSION,
  baseScore: Math.round(baseScore * 10) / 10,
  ratingPoints: Math.round(ratingScore * 10) / 10,
  reviewPoints: Math.round(volumeScore * 10) / 10,
  photoPoints: Math.round(visualScore * 10) / 10,
  infoPoints: Math.round(profileScore * 10) / 10,
  engagementPoints: Math.round(engagementScore * 10) / 10,
  competitivePoints: Math.round(competitiveScore * 10) / 10,
  franchiseBoostPoints: Math.round(franchiseResult.franchiseBoost * 10) / 10,
  rawScoreBeforeCap: Math.round(rawScore * 10) / 10,
  reviewReliabilityCap: reviewReliabilityCap,
  wasCapped: reviewReliabilityCapApplied,
  capReason: reviewReliabilityCapApplied && reviewReliabilityCap !== null
    ? `Only ${totalReviews} reviews. Score capped at ${reviewReliabilityCap} until you reach ${getNextReviewThreshold(totalReviews)} reviews.`
    : null,
  finalScore
};

// 3) meoWhy - Human-readable explanations
const meoWhy: string[] = [];

// Cap notification first (if applied)
if (reviewReliabilityCapApplied && reviewReliabilityCap !== null) {
  meoWhy.push(`🔒 Reliability cap applied: limited to ${reviewReliabilityCap}% because only ${totalReviews} reviews`);
}

// Rating with points
if (rating >= 4.8) {
  meoWhy.push(`Rating: ${rating.toFixed(1)}★ (excellent) → +${Math.round(ratingScore)} points`);
} else if (rating >= 4.0) {
  meoWhy.push(`Rating: ${rating.toFixed(1)}★ (good) → +${Math.round(ratingScore)} points`);
}
// ... etc for all components
```

### 3. Response Structure

Backend now returns:
```json
{
  "body": {

    "meoInputsUsed": { /* all raw GBP fields */ },
    "meoBreakdown": { /* points breakdown + cap info */ },

    "runId": "run_1234_abc",
    "debugStamp": "BACKEND_LIVE_2025-12-16T21:54:34.836Z"
  }

```

## Frontend Changes

### Updated `ScanResults.jsx`

Completely replaced the "Why this score" section to display the three explicit backend objects:

#### 1. **Why/Explanation Section**
Displays `meoWhy` array verbatim from backend:
```jsx
<ul className="space-y-2.5">
  {scanData.meoBackendData.meoWhy.map((reason, idx) => (
    <li key={idx}>
      <span>🔒 or •</span>
      <span>{reason}</span>
    </li>
  ))}
</ul>
```

#### 2. **Inputs Used Grid**
Displays `meoInputsUsed` in a 2-column grid with badges for booleans:
```jsx
<div className="grid grid-cols-2 md:grid-cols-3 gap-3">
  <div>Rating: {meoInputsUsed.rating.toFixed(1)}★</div>
  <div>Reviews: {meoInputsUsed.totalReviews}</div>
  <div>Photos: {meoInputsUsed.photoCount}</div>
  <div>Website: <Badge>{hasWebsite ? 'Yes' : 'No'}</Badge></div>
  <div>Phone: <Badge>{hasPhone ? 'Yes' : 'No'}</Badge></div>
  // ... etc
</div>
```

#### 3. **Score Breakdown List**
Displays `meoBreakdown` in a structured format:
```jsx
<div className="space-y-2">
  <div>Base: {baseScore}</div>
  <div>Rating: +{ratingPoints}</div>
  <div>Reviews: +{reviewPoints}</div>
  <div>Photos: +{photoPoints}</div>
  <div>Info: +{infoPoints}</div>
  <div>Engagement: +{engagementPoints}</div>
  <div>Competitive: +{competitivePoints}</div>
  {franchiseBoostPoints > 0 && <div>Franchise Boost: +{franchiseBoostPoints}</div>}
  <div>─────────</div>
  <div>Raw Score: {rawScoreBeforeCap}</div>
  {wasCapped && (
    <>
      <div>Cap: Applied ({reviewReliabilityCap})</div>
      <div className="alert">{capReason}</div>
    </>
  )}
  {!wasCapped && <div>Cap: Not applied</div>}
  <div>─────────</div>
  <div>Final: {finalScore}</div>
</div>
```

#### 4. **Error Handling**
If `meoWhy` is missing, shows red warning:
```jsx
{!scanData.meoBackendData.meoWhy && (
  <Card className="bg-red-50 border-red-300">
    <AlertCircle />
    Missing breakdown from backend
  </Card>
)}
```

## Key Design Principles

### ✅ Single Source of Truth
- Frontend **NEVER** generates any "why" text
- All explanations come from backend
- Frontend displays backend data **verbatim**

### ✅ Guaranteed Accuracy
- No client-side calculations
- No derived values
- No fallbacks or mocks (except explicit error state)

### ✅ Full Transparency
- Every input shown (`meoInputsUsed`)
- Every point calculation shown (`meoBreakdown`)
- Every reason explained (`meoWhy`)
- Cap logic clearly communicated

### ✅ Debuggability
- `runId` changes every scan
- `debugStamp` shows timestamp
- `scoringVersion` tracks algorithm version
- Console logs verify backend data
